We are beginning Phase 5 of the Core Platform CMS sprint: implementing reusable sections and saved components for the CMS builder.

CONTEXT
Admins need to create reusable content patterns from existing website sections and builder-created sections. This should work more like reusable saved sections/templates first, with a clean path toward broader synced components later.

IMPORTANT RULES
- Nondestructive only
- Reuse existing builder/block architecture
- Keep the first version practical and maintainable
- Do not overbuild a complex design-system engine
- Any reusable section containing image or background media must use the Cloudflare R2-backed CMS media workflow, not raw image URLs

PRIMARY GOAL
Allow admins to save and reuse section configurations across pages, based on the CMS builder architecture.

IMPLEMENTATION REQUIREMENTS
1. Inspect:
   - CMS builder/block architecture
   - seeded page content
   - likely reusable sections from existing Core Platform website pages
   - media handling for block content

2. Implement a reusable section/saved component system that supports:
   - save section from existing block/group of blocks
   - reuse section on other pages
   - edit reused content sensibly
   - choose whether reuse is copied/static or reference-based if that can be done safely
Start simple if needed.

3. Add a CMS Components / Reusable Sections area in admin.

4. Allow reusable items to store:
   - title/name
   - description
   - content JSON
   - category/type if useful
   - preview metadata if practical

5. Ensure media-bearing reusable sections preserve their R2-backed asset references correctly.

6. Keep this feature compatible with future global/synced section support, but do not overbuild that yet.

NONDESTRUCTIVE RULES
- No replacement of whole builder architecture
- No breaking seeded pages
- No raw image URL reintroduction
- No overengineered sync model if a simpler saved-section model is safer

VALIDATION CHECKLIST
1. Typecheck/check passes
2. Build passes
3. Admin can save reusable sections
4. Admin can insert reusable sections into pages
5. Media in reusable sections works correctly
6. Existing pages remain stable

REQUIRED OUTPUT
Provide:
1. Summary of reusable sections implementation
2. Files changed
3. Data/storage model notes
4. Any limitations or future recommendations
5. Docs update

DOCS UPDATE REQUIREMENT
Update docs to include:
- reusable section model
- save/reuse workflow
- media behavior for reusable components
- future synced/global component considerations
