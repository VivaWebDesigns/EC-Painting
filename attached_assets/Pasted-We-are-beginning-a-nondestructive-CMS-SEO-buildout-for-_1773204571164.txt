We are beginning a nondestructive CMS > SEO buildout for the Core Platform application.

CONTEXT
Core Platform now has a CMS area in Admin, but CMS > SEO is not yet built out. We need to create a real SEO control center for the public-facing site and blog. This should include both standard SEO settings and future-ready support for structured data.

IMPORTANT RULES
- Nondestructive only
- Preserve existing public routes, slugs, rendering, and blog behavior
- Do not break current metadata behavior if any already exists
- Keep implementation modular and aligned with the existing CMS/admin architecture
- SEO controls are for public-facing site content, not internal authenticated app screens
- Any image-related SEO fields must use Cloudflare R2-backed upload/dropzone or media-picker behavior, not pasted raw image URLs

PRIMARY GOAL
Build the CMS > SEO foundation including:
- CMS SEO overview page
- sitewide/global SEO settings
- shared SEO model/service structure
- future-ready integration points for structured data, redirects, sitemap visibility, and SEO audits

IMPLEMENTATION REQUIREMENTS
1. Inspect:
   - current CMS admin routing/navigation
   - current page/blog metadata handling
   - current public head/meta rendering approach
   - existing R2 media upload patterns
   - shared settings/config patterns
   - current blog schema and page schema
   - any current SEO-related fields or utilities already in the app

2. Build a new CMS > SEO admin area with a useful overview/dashboard.

3. Add sitewide/global SEO settings such as:
   - site name
   - default title suffix/pattern
   - default meta description
   - default canonical site URL/base URL support if appropriate
   - default social/open graph image
   - organization name
   - organization logo
   - social profile links
   - default robots/indexing preferences if appropriate
Do not overbuild unrelated marketing tools.

4. Use modular schema/service/storage patterns for SEO settings.

5. Ensure image-related SEO fields use Cloudflare R2-backed upload/dropzone flows, not raw URL inputs.

6. Make the architecture future-ready for:
   - per-page SEO
   - per-post SEO
   - structured data generation
   - technical SEO auditing
   - redirects and robots visibility

NONDESTRUCTIVE RULES
- No public slug changes
- No route rewrites
- No current page/blog rendering breakage
- No raw image URL inputs for SEO image fields
- No overbuilt enterprise SEO platform

VALIDATION CHECKLIST
1. Typecheck/check passes
2. Build passes
3. CMS > SEO route loads correctly
4. Admin can edit/save global SEO settings
5. Global SEO image fields use R2-backed upload/dropzone/media selection
6. Existing public pages and blog remain stable

REQUIRED OUTPUT
Provide:
1. Summary of SEO foundation added
2. Files changed grouped by area
3. Schema/settings notes
4. Architectural decisions made
5. Follow-up recommendations for per-page SEO and structured data
6. Docs update

DOCS UPDATE REQUIREMENT
Update docs to include:
- CMS SEO architecture overview
- global SEO settings model
- R2-backed SEO image handling
- future structured-data and technical SEO roadmap
