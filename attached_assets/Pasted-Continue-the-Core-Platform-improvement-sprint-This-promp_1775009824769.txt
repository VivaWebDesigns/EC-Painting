Continue the Core Platform improvement sprint.

This prompt is focused ONLY on therapist directory scalability and search correctness.

Problem to solve:
The therapist directory currently accepts filters like search, specialization, and language, but the server-side storage layer only partially filters data. The client fetches a large dataset and filters/sorts in memory. This must be moved to the database for scale.

Goal:
Implement proper server-side therapist directory filtering, sorting, and pagination while preserving the existing user experience as much as possible.

Tasks:
1. Inspect the current therapist directory flow, including:
   - directory.routes.ts
   - TherapistStorage.listProfiles
   - frontend therapist listing/filter components
2. Refactor the server-side query so filtering happens in SQL/Drizzle, including where applicable:
   - search
   - specialization
   - language
   - practiceMode
   - acceptingClients
   - approved/active status as required by business rules
3. Implement pagination response shape. Preferred shape:
   - items
   - total
   - page or nextCursor
   - pageSize
   - hasMore if useful
4. Support either:
   - page/pageSize pagination, or
   - cursor pagination
   Choose the approach that best fits the current architecture, but keep it simple and maintainable.
5. Update the frontend therapist directory to consume the new API response shape.
6. Keep client-side filtering only for temporary UI convenience if needed, but make the server query the source of truth.
7. Preserve existing directory cards, display behavior, and user-facing UX as much as possible.
8. Add validation for query params and sane defaults.
9. Add tests for:
   - filtering behavior
   - pagination behavior
   - empty results
   - combined filters

Nice-to-have if already supported by schema:
- Add sorting options in a safe way
- Prepare the query structure so geo-ranking can be added later

Rules:
- NON-DESTRUCTIVE
- Do not break existing public therapist browsing
- Avoid overengineering full-text search unless already simple to support
- Keep API response types shared and strongly typed
- If existing endpoint consumers expect old shape, update all relevant consumers together

Definition of done:
- therapist directory is filtered server-side
- pagination exists
- frontend consumes paginated response
- tests cover core filter combinations
- old in-memory full filtering is no longer the primary query logic
