We are beginning Phase 2 of the CMS > SEO sprint for Core Platform: implementing page-level and blog-post-level SEO field management.

CONTEXT
The CMS now has or is gaining SEO foundation and global settings. We need editors/admins to manage SEO fields for public pages and blog posts directly within the CMS and blog editing flows.

IMPORTANT RULES
- Nondestructive only
- Preserve public slugs and current rendering behavior
- Reuse current page/blog architecture where practical
- SEO controls should only apply to public-facing content
- Any image-related SEO/social fields must use Cloudflare R2-backed upload/dropzone or media picker, not raw URL entry

PRIMARY GOAL
Add page/post-level SEO controls including:
- SEO title
- meta description
- canonical URL
- noindex toggle where appropriate
- open graph title
- open graph description
- social/OG image
- basic preview/state visibility in admin

IMPLEMENTATION REQUIREMENTS
1. Inspect:
   - CMS Pages model/editor
   - blog post model/editor
   - current metadata rendering
   - current public head/meta generation patterns
   - R2 media handling

2. Add SEO fields to CMS pages and existing blog management, reusing current models where practical.

3. Ensure these fields can be managed in a common-sense place in the editing UI.

4. Add support for social/open graph images using the CMS media upload flow backed by Cloudflare R2.

5. Ensure rendering logic uses:
   - page-specific SEO values when present
   - blog-post-specific SEO values when present
   - sitewide defaults when page/post values are absent

6. Keep canonical and noindex logic clean and safe.

7. Do not introduce raw image URL fields for SEO images.

NONDESTRUCTIVE RULES
- No slug breakage
- No blog route breakage
- No public metadata regressions
- No raw image URL reintroduction

VALIDATION CHECKLIST
1. Typecheck/check passes
2. Build passes
3. Admin can edit page SEO fields
4. Admin can edit blog post SEO fields
5. SEO/social images use R2-backed upload/dropzone/media selection
6. Public page/post metadata renders correctly
7. Existing content remains stable

REQUIRED OUTPUT
Provide:
1. Summary of page/post SEO work
2. Files changed
3. Rendering/fallback notes
4. Any migration or backfill notes
5. Follow-up recommendations for structured data generation
6. Docs update

DOCS UPDATE REQUIREMENT
Update docs to include:
- page SEO fields
- blog SEO fields
- metadata fallback logic
- SEO image upload behavior
