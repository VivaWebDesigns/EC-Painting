We are beginning Phase 4 of the CMS > SEO sprint for Core Platform: building practical technical SEO tools in the CMS.

CONTEXT
Core Platform needs more than metadata fields. Admin users also need visibility into technical SEO factors and basic editorial controls such as redirects, sitemap awareness, robots/noindex visibility, and content-level SEO completeness.

IMPORTANT RULES
- Nondestructive only
- Preserve current public routes and SEO-critical slugs
- Do not accidentally deindex important content
- Keep tools practical and admin-friendly
- Do not overbuild enterprise crawler software
- Any image-related SEO fields shown here must still use the existing R2-backed upload/media flow

PRIMARY GOAL
Add practical technical SEO tools inside CMS > SEO, including:
- redirect management
- sitemap visibility/status
- robots/noindex visibility
- SEO audit tables for missing metadata and common content issues

IMPLEMENTATION REQUIREMENTS
1. Inspect:
   - routing patterns
   - sitemap generation if any exists
   - robots handling if any exists
   - page/post SEO fields
   - structured data engine
   - CMS page/blog/event inventories

2. Build a CMS > SEO audit view that can surface useful issues such as:
   - missing SEO title
   - missing meta description
   - missing social image
   - noindex pages
   - draft vs published visibility
   - potentially missing structured data opportunities where relevant
   - canonical gaps where applicable
Keep signals practical and non-noisy.

3. Add redirect management for common 301/302 use cases in a safe admin workflow.

4. Add sitemap visibility/status tools or at minimum a clear sitemap overview and what content is included.

5. Add robots/indexing visibility where practical, including noindex awareness from page/post settings.

6. Keep this tightly aligned with current routing and publishing behavior.

NONDESTRUCTIVE RULES
- No accidental slug changes
- No accidental deindexing
- No breaking redirects for current content
- No speculative technical SEO automation beyond safe admin tools

VALIDATION CHECKLIST
1. Typecheck/check passes
2. Build passes
3. SEO audit view loads and surfaces useful signals
4. Redirect management works safely
5. Sitemap visibility/status is clear
6. Noindex/indexing awareness is visible
7. Public pages remain stable
